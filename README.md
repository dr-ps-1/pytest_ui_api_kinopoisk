# final_proj: Кинопоиск: автоматизация UI и API тестов

Проект содержит автоматические тесты для веб-интерфейса и API сервиса «Кинопоиск» на основе финальной работы по ручному тестированию.

## Стек технологий
- **Python 3.14+** - основной язык программирования
- **Pytest** - фреймворк для написания и запуска тестов (поддержка маркеров `ui`/`api`)
- **Selenium + WebDriver Manager** - автоматизация браузера
- **Requests** - выполнение HTTP-запросов к API
- **Allure** - генерация отчётов о выполнении тестов
- **Python-dotenv** - загрузка переменных окружения из файла `.env`
- **PageObject** - паттерн для организации UI-тестов
- **Selenium-stealth** - маскировка автоматизации для снижения вероятности капчи
- **Fake-useragent** - генерация случайного User-Agent для браузера

**Форматирование кода**
- Код форматируется в соответствии с **PEP 8** (стиль написания кода на Python).
- Используются **docstrings** для документирования методов и функций.
- Все шаги теста размечаются с помощью `@allure.step` (для методов PageObject) или `with allure.step` (для блоков в тестах) для улучшения читаемости отчётов.
- Тесты имеют декораторы Allure:
  - `@allure.title` - заголовок теста.
  - `@allure.story` - название пользовательской истории (функциональности).

## Инструкция по запуску тестов и формированию отчёта Allure

1. **Установить необходимые зависимости**:
   ```bash
   pip install -r requirements.txt

2. Создать файл .env на основе .env.example и указать свои данные (особенно API-ключ).
Пример содержимого:
    UI_BASE_URL=https://www.kinopoisk.ru
    API_BASE_URL=https://api.poiskkino.dev
    API_KEY=your_api_key_here

3. Запустить тесты:
 - Все тесты: pytest
 - Только UI-тесты: pytest -m ui
 - Только API-тесты: pytest -m api

4. Запустить тесты с генерацией отчёта Allure:
   pytest --alluredir=./allure-results

Эта команда запустит все тесты (UI и API) и сохранит результаты в директорию ./allure-results.

Для запуска только определённой группы тестов используйте маркеры:
 - Только UI: pytest -m ui --alluredir=./allure-results
 - Только API: pytest -m api --alluredir=./allure-results

 # Инструкция по просмотру сформированного отчета Allure
После завершения тестов, чтобы просмотреть отчет, необходимо выполнить команду:
   ```bash
   allure serve ./allure-results
   ```
Эта команда запустит локальный сервер и откроет отчёт в браузере.

В отчёте Allure представлено:
Статистика по тестам: количество пройденных/упавших тестов.
Детали каждого теста:
 - Название теста (например, "Проверка отображения поля поиска на главной странице").
 - Шаги теста: открытие страницы, проверка капчи, закрытие всплывающих окон, поиск элемента, проверка результата.
 - Результат: успешное выполнение или ошибка с указанием ожидаемого и фактического результата.

# Примечания по стабильности UI-тестов
Сайт Кинопоиска активно использует средства защиты от автоматизации (капча, модальные окна). Для повышения стабильности реализованы следующие механизмы:
 - Маскировка браузера через selenium-stealth и случайный user-agent.
 - Автоматическое закрытие всплывающих окон (включая окно «Выберите, где продолжить»).
 - Загрузка сохранённых кук авторизации (если файл cookies.pkl существует). Для создания кук необходимо выполнить скрипт python save_cookies.py и войти в аккаунт вручную.
 - Ручное решение капчи - если она обнаружена, тест приостанавливается на 20 секунд, чтобы решить её вручную.

Несмотря на это, тесты могут иногда падать из-за внешних факторов. При локальном запуске рекомендуется следить за браузером и при необходимости решать капчу вручную. В большинстве случаев тесты проходят успешно.
